Live consent loops · witness receipts · provenance signals

Consent becomes live infrastructure.

LOOPtLOOP is a lightweight witness substrate for private authorization, live provenance, revocable consent-chain references, and consent-aware game participation. The current prototype proves one narrow but powerful claim: a private payload can remain private while a third party witnesses the hash, consent envelope, timing, and signature-bound receipt.

What this is

LOOPtLOOP turns consent and provenance into inspectable loops. A receipt witnesses origin. A status check can later determine whether a loop is still alive. A downstream actor can cite the chain without forcing hidden context into public view.

The service begins simply: hash locally, witness narrowly, verify publicly, keep private payloads private. The long destination is living consent-chain reliance: private context stays private, reliance stays accountable, consent stays revocable.

Glyph

🝳 means loop: relation with return.

LOOPtLOOP uses 🝳 as the public mark for living consent loops: a relation that can be opened, witnessed, relied upon, revoked, closed, and checked again. It is not merely recurrence. It is a path that returns with memory, consent state, and accountability.

For humans: 🝳 marks a bounded relation that remains alive enough to be checked.
For machines: 🝳 denotes a loop object whose origin, status, witness claims, and revocation surface should be inspected before reliance.
For artifacts: 🝳 points to provenance, consent conditions, and the witness record that makes the artifact admissible in context.

Human API notes

The API lives at https://api.looptloop.online/v0. The public source lives at github.com/bobrs/LOOPtLOOP.

The most important invariant: private payload stays private. Clients should hash private payloads locally and send only hash-bound witness metadata unless a future endpoint explicitly declares otherwise.

Core endpoints

Quick request shape

Create an authorization witness offer using a local hash. Do not send the private cleartext.

curl https://api.looptloop.online/v0/authorization-offers \
  -H "content-type: application/json" \
  -d '{
    "schema": "WITNESSKEY_AUTHORIZATION_OFFER_CREATE_0_1",
    "issuer": {"name": "WitnessKey", "origin": "https://witnesskey.online"},
    "event_type": "private_authorization_witnessed",
    "payload_hash": "sha256:<local-hash>",
    "payload_label": "optional human label",
    "declared_roles": ["human_authorizer", "ai_agent_actor"],
    "consent_prompt": "Do you consent to witness this private authorization by hash?",
    "return_url": "https://witnesskey.online/receipt-pending"
  }'

Operating principles

Witness claims are bounded. A witness can verify its witness function without claiming truth, legal identity, authorship, or action completion.

Consent loops are living. A receipt proves a loop was born; status and revocation determine whether it may still be relied upon.

Digital ego is not a profile. It is a living field of public, private, leaked, witnessed, revoked, fossilized, and forgotten micro-boundaries.
